# Введение

## Краткое описание проекта

Мобильная платформа для аквалангистов - это комплексная mobile-first система, объединяющая цифровой логбук, планировщик погружений, социальные функции, базу дайв-сайтов, бронирования дайв-туров, уведомления и административный контур.

Система ориентирована на использование в условиях нестабильной связи, поэтому для критичных пользовательских сценариев применяется offline-first подход: логбук, черновики планов, избранные дайв-сайты и часть пользовательских данных доступны локально на мобильном устройстве с последующей синхронизацией.

## Цель системы

Цель системы - предоставить дайверам, инструкторам и дайв-центрам единую платформу для хранения истории погружений, поиска и планирования погружений, взаимодействия внутри сообщества, организации курсов и туров, а также для безопасной модерации пользовательского контента.

Ключевые архитектурные цели:

1. Надежное хранение данных логбука.
2. Корректная синхронизация offline-изменений.
3. Поддержка мобильных сценариев на iOS и Android.
4. Масштабируемость социальных, поисковых и уведомительных функций.
5. Разделение доменных областей по DDD.
6. Возможность развития от modular monolith к выделенным сервисам без пересмотра доменной модели.

## Пользователи

1. Дайвер-любитель: ведет личный логбук, импортирует данные с дайв-компьютера, ищет дайв-сайты, планирует погружения, общается с другими дайверами, хранит сертификаты и отслеживает статистику.
2. Инструктор / дайв-гид: управляет учебными группами, фиксирует прогресс студентов по учебным погружениям, публикует курсы и дайв-туры, демонстрирует квалификацию.
3. Дайв-центр: ведет бизнес-профиль, публикует услуги, принимает запросы на бронирование, работает с отзывами и расписанием.
4. Администратор: модерирует пользователей, дайв-сайты, отзывы и социальный контент, блокирует нарушителей, контролирует качество данных и анализирует статистику использования приложения.

## Основные возможности платформы

1. Регистрация и авторизация пользователей.
2. Личный профиль пользователя, сертификаты и настройки единиц измерения.
3. Цифровой логбук с импортом данных с дайв-компьютеров.
4. Интерактивная база дайв-сайтов с поиском и фильтрацией.
5. Планировщик погружений с расчетом параметров, включая бездекомпрессионный предел для заданной глубины, и предупреждениями.
6. Социальные функции: публикации, комментарии, реакции, подписки, жалобы.
7. Бронирование дайв-туров и курсов, а также оплата premium-функций через внешнего платежного провайдера.
8. Уведомления через push, in-app, email и SMS при необходимости.
9. Административная модерация и аудит действий.

## Границы системы

Внутри границы платформы находятся:

1. Mobile App.
2. Backend Runtime как modular monolith.
3. Доменные модули User & Auth, Profile & Certificates, Logbook, Dive Sites, Dive Planner, Social, Booking, Notifications, Admin.
4. Primary Database.
5. Message Broker.
6. Search / Geo-indexing как производная read model.
7. Object Storage / CDN для медиа.
8. Background Workers.
9. Admin access contour.

За пределами границы платформы находятся:

1. Dive computer как физическое устройство, взаимодействующее с Mobile App через BLE, USB или vendor SDK.
2. Payment Provider.
3. Push Providers.
4. Maps / Geo Services.
5. Email/SMS Provider.

## Бизнес-ценность

Платформа объединяет разрозненные дайвинг-сценарии в единую цифровую среду: хранение истории погружений, поиск новых мест, планирование, социальное взаимодействие, профессиональные услуги и коммерческие бронирования. Для пользователей основная ценность заключается в сохранности данных логбука, удобстве offline-работы и доверии к планировщику. Для инструкторов и дайв-центров ценность выражается в управлении клиентами, расписанием, репутацией и бронированиями.
