### 8.9 ADR-009: Поиск и геопространственные данные для дайв-сайтов через отдельную поисковую проекцию

Статус: принято.

Контекст: пользователи ищут дайв-сайты по региону, сложности, типу, координатам, тегам, рейтингу и условиям. Карта и список должны отвечать быстро, особенно в mobile-first интерфейсе. При этом источником истины для дайв-сайтов остается Dive Sites Module.

Проблема / драйвер решения: геопространственный и полнотекстовый поиск имеют иной профиль нагрузки, чем транзакционное хранение карточек дайв-сайтов и модерации. Прямой поиск по основной модели может стать узким местом.

Рассмотренные альтернативы:

1. Выполнять все поисковые запросы напрямую к основной базе Dive Sites. Альтернатива проста, но плохо масштабируется для геофильтров, полнотекстового поиска и карты.
2. Полностью передать дайв-сайты внешнему картографическому сервису. Альтернатива снижает нагрузку на систему, но лишает платформу контроля над качеством, модерацией и доменной моделью.
3. Использовать отдельную Search / Geo-indexing проекцию, обновляемую из доменных событий Dive Sites.

Принятое решение: использовать отдельную поисковую и геопространственную проекцию для Dive Sites. Dive Sites Module остается источником истины для карточек, координат, статусов модерации и отзывов. Search / Geo-indexing хранит производную read model для быстрого поиска и карты. Конкретная технология поискового движка не фиксируется в ADR; она выбирается по требованиям к геоиндексации, полнотекстовому поиску, эксплуатации и стоимости.

Последствия: поиск становится быстрее и лучше масштабируется. Обновления индекса выполняются асинхронно после событий `DiveSiteApproved`, `DiveSiteUpdated`, `DiveSiteHidden`, `SiteReviewAdded`. UI должен учитывать eventual consistency: новый или обновленный сайт может появиться в выдаче с небольшой задержкой.

Риски / компромиссы: появляется необходимость переиндексации, мониторинга отставания индекса и обработки рассинхронизации. Внешние Maps / Geo Services не должны стать источником истины для статуса и качества дайв-сайтов.
